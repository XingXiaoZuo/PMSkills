# B端需求优先级评估与规划 (B2B Requirement Prioritizer)

## 技能背景与目标
基于“产品价值 = 客户价值 + 商业价值 - 研发成本”的综合权衡方法论，帮助产品经理将大量纷繁复杂的需求清单（文本或Excel）转化为有重点的、结构化的需求优先级清单。

> 理论支撑与案例参考：[如何有效判断需求优先级？](https://www.jianshu.com/p/c4916335c69f)

## 一、核心评估逻辑与公式

### 1. 核心公式
**产品价值 = 客户价值 + 商业价值 - 研发成本**
*(注：商业价值与客户价值在正向价值中的计算权重为 4:6)*

### 2. 客户价值判断 (权重 60%)
**公式：客户价值 = 需求客户数(权重30%) + 需求阻塞度(权重70%)**
- **需求客户数 (30%)**：
  - **前置动作**：必须在**全局解析完所有需求后**，识别同类/相似需求的总体出现频次。
  - **打分逻辑**：与全局需求清单对比，该同类需求被提及的次数越多，客户数得分越高。
- **需求阻塞度 (70%)**：
  - **强依赖关键字**：判断原始需求描述中是否包含“**卡实施**”、“**卡新签**”、“**卡续签**”、“**马上续费**”等严重阻塞商业流程的字样。
  - **打分逻辑**：若包含上述严重阻塞字样，则对应赋予高分；**若不包含任何阻塞字样，则该项得分为 0**。

### 3. 商业价值判断 (权重 40%)
相对宏观的价值判断，核心基于企业战略选择：
- **客群选择**：如专注中大型/超大型 vs 中小企业。
- **行业选择**：如专注餐饮连锁、制造医疗、或建筑房地产等。
- **赛道选择**：如聚焦 HR、ERP、财务赛道等。
- **重点方向判定规则**（重点关注插件化与AI）：
  - **插件化判定（强证据依赖）**：如果没有明确的对应依据，则不强行判断并给分。只有当需求描述或备注中明确包含“**可插件**”字眼时，才判定为符合插件化方向并给分；否则对应的插件化战略方向分值直接记为 **0**。
  - **AI 战略判定（场景推演）**：不能简单粗暴地依赖关键字（如“智能”或“AI”）匹配，而是需要**AI（你自身）进行深度语义分析与场景推演**，判断这个需求的痛点（如复杂的匹配、海量数据识别、自动推荐等）**是否可以通过 AI 的方式更高效地迭代解决**。如果能，则判定符合 AI 战略并给分；如果纯粹是固定的业务逻辑流转或字段增删，则不符合。

### 4. 研发成本 (负向项)
需要**AI（你自身）基于需求语义理解**进行开发与测试成本的预估判断。将研发成本分为 5 个等级，等级越高，对产品净价值的扣分/折损越大：
- **极低成本（1级）**：文案修改、简单字段增加、已有逻辑的轻微调整。
- **低成本（2级）**：简单的页面交互优化、已有接口的入参/出参扩展、不涉及历史数据清洗的单表 CRUD。
- **中等成本（3级）**：新增独立的小模块/页面、涉及少量跨模块交互、需要新增较多测试用例的常规迭代。
- **高成本（4级）**：涉及底层数据模型调整、强依赖外部系统/第三方接口深度对接、需要处理大量历史数据兼容、复杂的权限/审批流改造。
- **极高成本（5级）**：核心计算引擎重构（如算薪/排班底层重写）、全系统级别的架构升级、极其复杂的算法模型（大模型训练/微调）落地。

## 二、执行约束与工作流 (⚠️必须严格遵守)

**第一步：校验战略信息（拦截校验）**
- **强规则**：如果不清楚客户的战略选择，无法判断商业价值，则**必须首先要求用户输入企业的 OKR 或战略方向**（如客群选择、行业选择或重点方向等）。
- **阻断**：在用户未提供战略方向前，**绝对不可**盲目输出需求优先级。

**第二步：需求解析与一对一评估**
- 接收用户的需求清单（直接输入文本或解析上传的Excel）。
- **强制不遗漏且不合并**：必须 100% 解析清单中的所有数据，**禁止合并需求**，严格保持与原始数据一对一的对应关系，绝不能遗漏任何一条原始数据。
- 提取评估所需的关键字段（如原优先级、需求关键字、需求痛点/期望效果、备注等）。

**第三步：优先级打分定级**
- 结合确认后的战略方向，对每一条独立的需求应用上述核心公式进行打分。
- 将最终得分映射为 P0 到 P4 的 5 个优先级等级，其中 **P0 最优先，P4 最不优先**。

**第四步：输出需求清单 (Excel表/结构化表格格式)**
输出一份结构化的表格（若在对话框中可用 Markdown 表格替代展示，若涉及大量数据推荐直接生成 Excel 文件供下载），必须包含以下核心列：
- **最新需求优先级** (评估后得出的 P0 ~ P4)
- **原优先级** (取自原需求表格里的优先级内容)
- **需求关键字** (取自原需求表格里的内容)
- **需求说明** (基于原表格里的痛点描述、期望效果提炼的说明)
- **备注** (取自原需求表格里的备注内容)
- **价值分说明** (客户价值、商业价值与研发成本的得分简述及计算数值)
- **判断依据** (一句话说明为什么给出这个分数，如“包含卡新签字样，且符合插件化方向”)
