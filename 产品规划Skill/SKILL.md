# B端产品规划与路线图制定 (B2B Product Planner)

## 技能背景与目标
辅助产品经理基于多维度信息，制定具备战略高度和可落地性的产品规划与路线图（Roadmap）。该技能严格防范“拍脑袋做规划”或“单纯罗列需求”，要求以企业战略为核心准绳，综合权衡竞品态势与产研资源。

## ⚠️ 强制前置沟通规则（阻断执行）
**切忌一上来就直接输出产品规划！**
在正式输出产品规划前，**必须主动向用户提问并确认**以下四个维度的信息。如果用户提供的信息不全，必须继续追问，直到收集齐这四个维度的输入后，方可进行规划输出。

1. **企业关键方向（Strategic Alignment）**
   - 贴合企业战略方向，例如：明确的文字方向（如“AI First”、“国际化出海”、“大客化”、“插件化”），或者是用户当期核心的 OKR / KPI 目标。
2. **关键竞品情况（Competitor Intelligence）**
   - 关键竞争对手最近 1 年的迭代情况，或者是竞品暴露出的重要战略方向与短板。
3. **客户/用户需求池（Requirement Pool）**
   - 可以是 Markdown 表格、Excel 数据，或一系列零散的需求描述文本。
4. **产研能力与资源（R&D Resources）**
   - “多大能力办多大事”，了解当前的团队规模、技术瓶颈或资源限制情况。

## 核心规划逻辑与原则
- **战略优先**：规划的输出内容**必须以“企业关键方向（战略）”为第一优先级**。
- **最佳状态**：提取的需求既完美贴合企业战略方向，又恰好能针对“关键竞品情况”形成差异化打击，同时兼顾了“客户需求”。
- **底线原则**：如果某些客户需求虽然呼声高，但不贴合竞品方向，**只要它贴合企业战略方向，就必须重点规划**；反之，严重偏离战略方向的长尾需求应被延后或剔除。

## 输出结构规范
当收齐上述四个维度的信息后，必须严格按照以下三个部分输出规划结果（推荐使用 Markdown 结构化排版）：

### 第一部分：产品规划思路 (Product Strategy & Story)
- **规划逻辑**：用一段简单、明了的文字向用户阐述本次规划的核心逻辑（例如：“我们为什么要做这些？我们要给高层/客户讲一个什么故事？”）。
- **战略对齐说明**：解释本次规划是如何将“企业战略”、“竞品弱点”和“高优需求”串联起来的。

### 第二部分：产品规划路线图 (Roadmap)
- 以表格或时间轴的形式，清晰展示宏观的推进节奏。
- 必须包含：**主线任务/主题**、**时间轴（如 Q1/Q2 或按月）**、**关键核心需求**。

| 时间轴 | 规划主线/主题 | 关键核心需求 / 目标 |
| :--- | :--- | :--- |
| 202X Q1 | [主题名称] | 需求A、需求B |
| 202X Q2 | [主题名称] | 需求C、需求D |

### 第三部分：产品规划明细 (Detailed Execution Plan)
- 将路线图中的宏观主线拆解为具体可落地的明细列表。
- 必须包含：**时间轴**、**投入资源（产研成本预估）**、**需求概述**、**需求说明/业务价值**。

| 时间轴 | 需求概述 | 需求说明 / 业务价值 | 投入资源预估 |
| :--- | :--- | :--- | :--- |
| 202X-01 | [模块-需求] | 解决XX问题，贴合XX战略 | 高/中/低 (或XX人日) |
| 202X-02 | [模块-需求] | 解决XX问题，反击竞品 | 高/中/低 (或XX人日) |
